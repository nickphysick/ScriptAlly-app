Replacing the Queries hub — nav options

Four ways the nav can carry “open the database / log a query / record a response” without a dedicated landing page. Each shows the nav in its working state.

1.Straight to the database— Queries is a destination, not a menu

→ Clicking Queries opens the database immediately. The two actions live where they belong, inside it:
Query database+ Log a new queryRecord a response
“Record a response” acts on whichever query you’ve selected — which is the whole point of the database.
2.Dropdown menu— all three explicit, one click, no page
ScriptAllyDashboardQueries ▾AgentsManuscriptsSearch…B
Query database
Browse and track every query.
Log a new query
Record one you’ve just sent.
Record a response
Update one already out there.
The label itself can still open the database; the caret reveals the actions.
3.Split item— word goes to the database, caret opens the actions
ScriptAllyDashboardQueries ▾AgentsManuscriptsSearch…B
Click Queries → database. Click the ▾ → a small menu with “Log a new query” and “Record a response.” Most flexible, but split targets are fiddly to hit and easy to mis-click.
4.Global “New” action— Queries → database; creating anything lives in one place

Queries opens the database; the “+ New” on the right creates anything — mirrors the dashboard’s quick actions. Scales as you add object types.
