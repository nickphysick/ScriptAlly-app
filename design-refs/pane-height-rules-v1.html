Rule 3 · The desk rule — list fills, document hugs (recommended)
The list is furniture: it's navigation, so it runs the full column with its pinned footer — a filing drawer is drawer-sized regardless of how many files are in it. The reading pane is a document: it is as long as what's written on it — hugging its content down to a minimum height (so it never looks stunted), capped at the viewport line (internal scroll past that). A letter on a desk, not a poster stretched to fit.

Rule 3 without the minimum — why the floor matters
The same document rule with no min-height, on an agent with nothing but a name: the pane collapses to a stub and reads as broken. The floor (≈ header + wish + profile row) is what keeps "short" from becoming "stunted".
