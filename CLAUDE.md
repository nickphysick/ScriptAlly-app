## Multi-stream protocol (several Claude Code sessions share this repo, all on main, at once)
- **Identity gate:** any cross-session note (resume/sync/follow-up prompts) starts with self-identification from the session's OWN history; a note that reaches the wrong stream gets its facts verified, not obeyed — they may be stale (a "tsc is red" another session already fixed; a "pending deploy" that landed an hour ago).
- **Resume = re-read:** after any parking, in-context copies of shared files are stale by definition — re-read from disk before editing. Never edit another stream's uncommitted WIP (live lost-update risk); leave a handover note (BUILD-REPORT/CLAUDE.md/commit message) instead.
- **Commits:** explicit-path staging + `git commit --only -- <paths>` (race-proof against the shared index); `git diff --cached --name-only` must list only your stream's paths. No `git stash` / `checkout --` / `restore` / `clean` on anything, ever. Gates (tsc + production build + full Vitest) green before every commit — with `set -o pipefail` (piping to tail otherwise masks a red tsc, which also emits JS despite errors). If a gate is red from ANOTHER stream's untracked WIP, verify in an isolated HEAD worktree + only your files.
- **Reports:** per-stream report files (the `LOCATION_BUILD_REPORT.md` / `DISCOVER_PHASE1_REPORT.md` pattern). `BUILD-REPORT.md` is the legacy shared file — append-only, and a run note rides the SAME commit as its code (no floating hunks left in shared tracked files).
- **Test counts move several times a day across streams** (523 → 538 → 542 within 5 Jul alone) — record the suite figure fresh at commit time; never copy it from an earlier note or chat.

## Agent location / territory
- **Model:** the ONLY stored location data is `agent.country` (ISO 3166-1 alpha-2, e.g. `"GB"`) + optional `agent.city`, and `user.homeCountry` (ISO alpha-2). Everything else — the flag, the display name, the home-vs-foreign distinction, any market count — is **derived at read time** from those codes. Nothing territory-derived is stored on the agent, and **nothing is denormalised onto queries** (`recomputeQuery` is inert to location — recon-confirmed).
- **ISO alpha-2 is canonical.** Legacy full-name values (the name-based `COUNTRIES` in `agentOptions.ts` + the undeployed v12 Edit Agent drawer) are **tolerated at read time only** — `territory.ts` reverse-maps them to codes; consumers migrate to writing codes in Prompt 2.
- **Never `null`/`""`.** "Not set" always means the **key is omitted**. The `isValidAgent`/`isValidUser` rules reject a literal null (`data.x is string`). The signup seed (`homeCountrySeed()`) and quick-add both omit rather than write empty.
- **Single source of truth = `src/lib/territory.ts`** (Prompt 1): `COUNTRIES_ISO`, `QUICK_PICKS` (`GB US IE CA AU NZ`), `countryName`, `flagFor` (→ `flag-icons` class `"fi fi-gb"`), `isHomeMarket`, `detectHomeRegion` (browser locale region + tz fallback, **never IP**), `getHomeCountry` (stored ?? detected ?? `GB`), `homeCountrySeed`. Flags = the whole `flag-icons` SVG set, served as static assets (CSS-referenced, out of the main JS chunk), addressable by ISO code — **not** emoji (Windows renders flag emoji as letter-pairs).
- **`homeCountry`** seeded silently from the browser locale at signup, no onboarding step, no backfill (`getHomeCountry` covers absent users at read time), editable later in settings (Prompt 2/3).
- **Rules edited, NOT deployed.** The `homeCountry` clauses (`isValidUser` + the user-update allowlist) ride the **same `firestore:rules` deploy** as the already-in-file-but-undeployed `country`/`city` agent allowlist — **nothing location-related works in prod until that deploy lands.** Claude Code must not deploy.
- **Prompt 1 done (foundations, on `main`):** `territory.ts` + tests (`territory.test.ts`, `homeCountryRule.test.ts`), `flag-icons` installed, `User.homeCountry`, the two rule edits, the signup seed. **Prompt 2/3 (not started):** the Add-Agent Location section + single-select country picker (adapt `GenreCombobox`), `EditAgentDrawer`/`agentOptions.COUNTRIES` migration to ISO, the agent-DB territory marker (home = sage pill / foreign = flag-only-with-hover), and the `AccountSettings` Preferences homeCountry editor.
