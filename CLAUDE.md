## Conventions & invariants
- **QueryStatus:** always use the exact `QueryStatus` enum strings (e.g. `"Partial Requested"`, `"Revise & Resubmit"`). Never camelCase or ad-hoc variants.
- **Undo:** undo must delete the original activity records this action created/modified ŌĆö never append compensating entries.
- **Response counting:** each query counts as at most one response, regardless of pipeline stage.
- **Canonical nav active treatment (app-wide):** primary-nav links are **black `#241c15` text in every state** ŌĆö never burgundy. ## Agent location / territory
- **Model:** the ONLY stored location data is `agent.country` (ISO 3166-1 alpha-2, e.g. `"GB"`) + optional `agent.city`, and `user.homeCountry` (ISO alpha-2). Everything else ŌĆö the flag, the display name, the home-vs-foreign distinction, any market count ŌĆö is **derived at read time** from those codes. Nothing territory-derived is stored on the agent, and **nothing is denormalised onto queries** (`recomputeQuery` is inert to location ŌĆö recon-confirmed).
- **ISO alpha-2 is canonical.** Legacy full-name values (the name-based `COUNTRIES` in `agentOptions.ts` + the undeployed v12 Edit Agent drawer) are **tolerated at read time only** ŌĆö `territory.ts` reverse-maps them to codes; consumers migrate to writing codes in Prompt 2.
- **Never `null`/`""`.** "Not set" always means the **key is omitted**. The `isValidAgent`/`isValidUser` rules reject a literal null (`data.x is string`). The signup seed (`homeCountrySeed()`) and quick-add both omit rather than write empty.
- **Single source of truth = `src/lib/territory.ts`** (Prompt 1): `COUNTRIES_ISO`, `QUICK_PICKS` (`GB US IE CA AU NZ`), `countryName`, `flagFor` (ŌåÆ `flag-icons` class `"fi fi-gb"`), `isHomeMarket`, `detectHomeRegion` (browser locale region + tz fallback, **never IP**), `getHomeCountry` (stored ?? detected ?? `GB`), `homeCountrySeed`. Flags = the whole `flag-icons` SVG set, served as static assets (CSS-referenced, out of the main JS chunk), addressable by ISO code ŌĆö **not** emoji (Windows renders flag emoji as letter-pairs).
- **`homeCountry`** seeded silently from the browser locale at signup, no onboarding step, no backfill (`getHomeCountry` covers absent users at read time), editable later in settings (Prompt 2/3).
- **Rules edited, NOT deployed.** The `homeCountry` clauses (`isValidUser` + the user-update allowlist) ride the **same `firestore:rules` deploy** as the already-in-file-but-undeployed `country`/`city` agent allowlist ŌĆö **nothing location-related works in prod until that deploy lands.** Claude Code must not deploy.
